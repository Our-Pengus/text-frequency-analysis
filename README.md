# 텍스트 빈도 분석기 기획서

## 프로젝트 개요

텍스트에서 단어의 출현 빈도를 분석하여 가장 자주 사용된 단어를 파악할 수 있는 웹 애플리케이션입니다. 대용량 텍스트도 빠르게 처리할 수 있도록 WebAssembly 기술을 활용합니다.

---

## 핵심 기능 기획

### 1. 텍스트 입력
- **요구사항**: 다양한 길이의 텍스트 입력 지원
- **제약사항**: 
  - 최대 입력 길이 제한 (메모리 고려)
  - 특수문자, 이모지 처리
- **UX 고려사항**: 
  - 텍스트 영역 크기 조절 가능
  - 입력 중 실시간 글자 수 표시

### 2. 단어 분리 및 정규화
- **규칙 정의**:
  - 공백, 구두점 기준으로 단어 분리
  - 대소문자 구분 없음 (예: "The"와 "the"는 동일)
  - 하이픈(-), 아포스트로피(') 포함 단어 처리
  - 숫자 포함 단어 처리 방식 결정
- **예외 처리**:
  - 연속된 공백 처리
  - 줄바꿈 처리
  - 특수문자만 있는 경우 제외

### 3. 빈도 계산 및 정렬
- **계산 방식**: 각 단어의 출현 횟수 카운트
- **정렬 기준**: 빈도 내림차순 (가장 많이 나온 단어가 위)
- **동점 처리**: 빈도가 같을 경우 알파벳순 또는 입력 순서 유지

### 4. 결과 표시
- **전체 분석 모드**:
  - 모든 단어의 빈도 표시
  - 스크롤 가능한 긴 목록
- **상위 N개 모드**:
  - 사용자가 지정한 개수만큼만 표시 (기본 10개)
  - 빠른 인사이트 제공
- **결과 형식**:
  - 순위, 단어, 빈도 수를 표 형태로 표시
  - 시각적으로 구분하기 쉬운 디자인

### 5. 성능 최적화
- **처리 속도 목표**:
  - 1KB 텍스트: 1초 이내
  - 100KB 텍스트: 5초 이내
  - 1MB 텍스트: 30초 이내
- **사용자 경험**:
  - 분석 중 로딩 상태 표시
  - 진행률 표시 (선택사항)
  - UI 블로킹 방지
